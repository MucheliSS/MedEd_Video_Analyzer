# ER Simulation Video Analyzer

A privacy-first Streamlit application that analyzes emergency room simulation videos (sim wars, mock resuscitation drills) against evaluation rubrics using a local Vision Language Model (VLM) via LM Studio. No data leaves your machine.

## Features

- **Video frame extraction** — intelligent sampling using interval-based + scene-change detection (OpenCV)
- **Audio transcription** — local speech-to-text via Whisper (optional, requires ffmpeg)
- **Character tracking** — identifies and tracks team members across frames by visual descriptors (clothing, position, role)
- **Flexible rubrics** — upload your own (JSON/CSV), auto-generate from video content, or use the built-in ER resuscitation rubric
- **Rubric scoring** — each item scored as Yes / Partial / No / N/A with evidence citations and timestamps
- **Narrative report** — structured debrief summary (strengths, areas for improvement, key moments, recommendations)
- **Export** — download reports as JSON or plain text

## Prerequisites

- **Python 3.11+**
- **LM Studio** running locally with a vision-capable model loaded (e.g., Gemma 4)
  - Default API endpoint: `http://localhost:1234/v1`
- **ffmpeg** (required for audio transcription; optional if you skip audio)

## Usage

1. Start LM Studio and load a vision-capable model (e.g., Gemma 4)
2. Run the app:

```bash
streamlit run app.py
```

3. In the browser UI:
   - **Upload** a simulation video (mp4, avi, mov, mkv, webm — up to 1 GB)
   - **Choose a rubric**: upload your own, auto-generate, or use the default
   - **Click "Analyze Video"** and wait for the pipeline to complete
   - **Review results** across tabs: Rubric Scores, Narrative Report, Team Members, Timeline, Export

## Rubric Formats

### JSON

```json
{
  "title": "My Custom Rubric",
  "description": "...",
  "categories": [
    {
      "name": "Category Name",
      "items": [
        {"id": "C1", "text": "Observable behavior to evaluate", "weight": 2}
      ]
    }
  ]
}
```

See `sample_rubric.json` for a complete example. Sample templates are also downloadable from within the app.

### CSV

```csv
category,id,text,weight
"Team Communication",TL1,"Closed-loop communication used",2
"Primary Survey",PS1,"Airway assessed appropriately",2
```

## Analysis Pipeline

The analysis runs in five sequential steps:

1. **Frame extraction** — samples frames at configurable intervals; detects scene changes via histogram comparison
2. **Character identification** — two-round process: VLM identifies team members by persistent visual features (clothing, body position, build), then verifies against a different set of frames for consistency
3. **Frame-by-frame analysis** — batched VLM calls describe clinical actions, communication, and concerns using the character labels
4. **Rubric scoring** — VLM evaluates each rubric item against accumulated evidence from all frames + audio transcript
5. **Narrative report** — generates a structured debrief summary

## Configuration

### Sidebar controls (runtime)

- **Model selection** — picks from models loaded in LM Studio
- **Frame interval** — seconds between sampled frames (lower = more detail, slower)
- **Max frames** — cap on total frames analyzed
- **Audio transcription** — toggle on/off

### `config.py` (defaults)

| Setting | Default | Description |
|---------|---------|-------------|
| `LM_STUDIO_BASE_URL` | `http://localhost:1234/v1` | LM Studio API endpoint |
| `FRAME_INTERVAL_SEC` | 3 | Seconds between frame samples |
| `MAX_FRAMES` | 60 | Maximum frames to extract |
| `MAX_FRAMES_PER_BATCH` | 4 | Frames per VLM API call |
| `WHISPER_MODEL` | `base` | Whisper model size (`tiny`/`base`/`small`/`medium`/`large`) |

### `.streamlit/config.toml`

Upload size limit is set to 1 GB. Edit `maxUploadSize` to change.

## Output

Reports are saved to the `outputs/` directory with both:

- `report.json` — full structured data (rubric scores, frame analyses, transcript, character info)
- `report.txt` — human-readable text report for printing or sharing

## Transparency

When the rubric is auto-generated by the AI model (rather than uploaded by a human evaluator), this is clearly flagged in:

- The Streamlit UI (warning banner)
- The exported text report (header notice)
- The JSON report (`"auto_generated": true` field)

## Privacy

All processing happens locally:

- Video frames are analyzed by your local VLM in LM Studio
- Audio is transcribed by a local Whisper model
- No API keys, cloud services, or external network calls are used
- Uploaded videos and extracted frames stay in the local `uploads/` directory
